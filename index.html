HTML = Hyper Text Markup Language

  • HTML = content
  • HTML skeleton shortcut ! + tab in VS cde
  • üê® HTML skeleton explained:
    1. !DOCTYPE must be present and informs that this is an HTML document
    2. html start and html end tag define the start of an html document
    3. language has to be specified in the opening html tag
    4. head element is the susbequent, and it contains all the infromation for the browser which are not visible to users
    5. meta charset tag defines the character set
    6. title and also any links to for eg stylesheet and others
  • Head is followed by the body element, which represents the visible content shown to the user.
  • Body can contain main section which can furtehr contain header, navigation, sections, articles etc etc... (checkout MDN on these if you get stuck)
  • It is good prctice to use HTML semantic elements rather than 'neutral' ones such as div and span. Semantic elements enable better accessiblity for both users and developers (SEO for eg...)
  • Elements can have id or class which is then useful for styling them. Essentially id is a more specific element than a class and will always take precedence. We can add more classes to an element if we want to be specific on some styling parts
  • KEEP IT SIMPLE AS POSSIBLE

Git

  • Terminal: cd = change directory, cd - = previous working directory, cd .. = return to starting directory, mkdir = create a new directory, pwd = print working directory, touch file-name = create a new file, code . = open VS code, git status = current branch, git restore = restore code to the last saved point, git clean -xdf = return to the initial code
  • üí° Create a local repo and clone it to git:
    1. mkdir the folder in terminal
    2. cd into the above mentioned folder
    3. code . to open VS code
    4. create README.md
    5. go to Github and create a repo with the name only
    6. copy the first shown code into the terminal and hit enter
    7. there you go! now just make sure to add, commit and push changes
  • üí° Create a github repo and clone it locally:
    1. create github repo + readme.md and/or license but not needed for the purpose of this course, for now anyway
    2. mkdir folder on a local machine using terminal
    3. cd into above mentioned folder
    4. git clone and copy SSH code into the terminal from github and hit eneter
    5. DONE! do not forget to add, commit and push
  • Check out Git cheat sheet in case you get stuck with these
  • When working with branches on a team project. CREATE A NEW BRANCH: git checkout -b feature/branch-name
  • subsequently add, commit and push
  • when pushing ensure to specify the branch, i.e. git push feature/index.html
  • ENSURE NEW BRANCH IN SYNC WITH BASE BRANCH git pull origin main
  • PUSH into github repo: git push origin feature/file-name
  • FINAL STEP ALWAYS!! to update local repository: git checkout main git pull origin main

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is a single variable that is used to hold a group of data.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. 'if' statements are used for flow to be interrupted
  • A for loop contains 3 important statements: 1. starting point for our loop. Since we want to start at the beginning of the array - variable with a value of 0: let x = 0; 2. the condition As long as the number in the variable x is less than the length of the array, the loop will keep running: x < shapes.length; 3. what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1
  • Functions are the building blocks of JS. When we create a function, we are defining a block of code that performs a specific task and giving it a unique name. However, just defining a function does not mean it will execute in our code automatically --> to execute a function, we must call it in our code.
